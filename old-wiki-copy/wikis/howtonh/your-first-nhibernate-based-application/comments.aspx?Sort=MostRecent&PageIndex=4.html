By: anyeone Posted on 07-08-2011 14:31

Thank you for the work you did on this tutorial.  The tutorial is well written and clear but unfortunately it skips several steps that may be second nature to people who work with Castle already, but not for everyone.  For reference I am using VS 2010.  I have NUnit installed on my machine already but not Castle, and I downloaded a fresh install of NHibernate today (first time on my machine).

These are the issues I had with the tutorial steps as written, and what I did to resolve them.

1) Castle.DynamicProxy2.dll is not included with the NHibernate install as of today's date.  I downloaded Castle to acquire it.

2) SchemaExport.Execute does not have a four-boolean overload in today's version.  There is a three boolean overload and a couple that have IO writers in them.  Unfortunately I do not know which booleans in your sample correspond to which actual properties - it would be helpful to clarify this so I could determine how I should be setting these to best follow the example.  I just put (false, true, false) but I have no idea if that is what was intended.

3) After I made the Product properties virtual, I still got an error trying to run the test for Add():

NHibernate.Bytecode.ProxyFactoryFactoryNotConfiguredException:  The ProxyFactoryFactory was not configured. Init

Failed: TestfixtureSetup failed in ProductRepository_Fixture

I googled the error and found a reference to an nhibernate config file setting for

<property name='proxyfactory.factory_class'>NHibernate.ByteCode.Castle.ProxyFactoryFactory, NHibernate.ByteCode.Castle</property>

I added this to the nhibernate config file (in both the project and test project), which of course required that I add NHibernate.ByteCode.Castle.dll to the project references (this dll was not mentioned or pictured in the tutorial).  You really should these steps in for people less adept at troubleshooting than myself ;)

My last comment is totally a nitpick  but a good unit test only has one assert in it:  several of your unit tests have several asserts.  The later ones won't run if the former fail.  Maybe just leave one assert per test if you don't want to show multiple tests (which would be overkill for a tutorial), so that folks new to unit testing don't pick up bad habits :)

I hope you'll take these comments as constructive as they are intended to be - overall it was a very helpful tutorial and I appreciate your effort!
